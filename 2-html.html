¿QUÉ ES HTML?

HTML significa "HyperText Markup Language" (Lenguaje de Marcado de Hipertexto) y es el lenguaje estándar que se utiliza para estructurar y presentar el contenido de las páginas web. Utiliza etiquetas (tags) para organizar elementos como texto, imágenes, videos y enlaces, y sirve como el esqueleto fundamental que los navegadores web interpretan para mostrar el contenido a los usuarios. Componentes y función principal

  • Lenguaje de marcado: A diferencia de los lenguajes de programación, HTML no ejecuta lógica; su función es marcar y describir la estructura del contenido.
  • Estructura el contenido: Define cómo se organizan los elementos en una página web. Por ejemplo, usa etiquetas para crear encabezados, párrafos, listas, tablas y más.
  • Es el esqueleto de la web: Proporciona el marco sobre el cual se construyen los sitios web, asegurando que la información sea visualmente atractiva, accesible y funcional en diferentes dispositivos.
  • Utiliza etiquetas: El código HTML se compone de etiquetas de apertura y cierre que delimitan el contenido. Por ejemplo, "< p> este es un párrafo < /p>".
  • Habilita los enlaces (hipertexto): Permite conectar documentos entre sí o diferentes secciones dentro del mismo documento, utilizando etiquetas especiales como "< a>".
  • Se complementa con otros lenguajes: HTML trabaja en conjunto con otros lenguajes. CSS se encarga de los estilos visuales, y JavaScript añade interactividad.

    ¿Cómo funciona?

  • El código HTML define la estructura de la página.
  • El navegador web lee este código.
  • El navegador interpreta las etiquetas y las utiliza para mostrar el contenido en un formato organizado y visual.

    • ESTRUCTURA BASICA DE HTML

    La estructura básica de HTML se basa en la declaración < !DOCTYPE html> y las etiquetas principales < html>, < head>, y < body>. El < head> contiene información no visible de la página como el título (< title>) y metadatos, mientras que el < body> alberga todo el contenido visible para el usuario.

  • < !DOCTYPE html>: La primera línea del documento, que indica al navegador que se trata de un documento HTML5.
  • < html>: La etiqueta raíz que envuelve todo el contenido de la página.
  • < head>: La sección de la cabecera, donde se incluye información sobre la página que no es directamente visible, como:
  • < title>: El título de la página que aparece en la pestaña del navegador.
  • Meta datos, estilos CSS y scripts de JavaScript.
  • < body>: La sección del cuerpo, que contiene todo el contenido que se mostrará al usuario, como texto, imágenes, enlaces, etc.

    • Ejemplo de estructura básica:

    < !DOCTYPE html>

    < html>

    < head>

    ....< title>Mi Primera Página< /title>

    < /head>

    < body>

    ....< h1>¡Hola, Mundo!< /h1>

    ....< p>Este es un párrafo de ejemplo.< /p>

    < /body>

    < /html>

    • COMO FUNCIONAN

    Estructura y contenido principal

  • < html>: Define todo el documento HTML.
  • < head>: Contiene metadatos y enlaces a otros recursos, como CSS y JavaScript.
  • < title>: Establece el título de la página que se muestra en la pestaña del navegador.
  • < body>: Contiene todo el contenido visible de la página web.

    • Títulos y texto

  • < h1> - < h6>: Definen títulos y subtítulos. < h1> es el más importante y < h6> el menos importante.
  • < p>: Define un párrafo de texto.
  • < strong>: Indica que el texto tiene una fuerte importancia, visualmente se representa en negrita.
  • < em>: Indica énfasis en el texto, se suele mostrar en cursiva.

    Multimedia

  • < img>: Inserta imágenes en la página web.
  • < iframe>: Se utiliza para incluir contenido de otra página web o fuente externa.

    Listas

  • < ul>: Crea una lista desordenada (viñetas).
  • < ol>: Crea una lista ordenada (numerada).
  • < li>: Define cada elemento dentro de una lista (ordenada o desordenada).

    Agrupación y diseño

  • < div>: Crea un contenedor genérico para agrupar elementos y aplicar estilos a secciones de la página.
  • < span>: Agrupa elementos en línea, útil para aplicar estilos a una parte específica de un texto.

    Enlaces y formularios

  • < a>: Crea un hipervínculo para enlazar a otra página o recurso.
  • < form>: Define un formulario en la página web.
  • < label>: Asocia una etiqueta de texto a un campo de formulario, mejorando la accesibilidad.

    Tablas

  • < table>: Define una tabla.
  • < tr>: Define una fila en una tabla.
  • < th>: Define una celda de encabezado en una tabla.
  • < td>: Define una celda de datos en una tabla.

    • CAMBIOS HISTORICOS

    Los cambios históricos clave en HTML incluyen su origen en 1991 con el HTML 1.0 de Tim Berners-Lee, la estandarización de características en las versiones 2.0 y 3.2, y la introducción de la separación entre contenido y presentación en HTML 4 en 1997. El avance más significativo llegó con HTML5 en 2014, que incorporó soporte nativo para multimedia, gráficos, y nuevas etiquetas semánticas, haciendo el desarrollo web más moderno y accesible.

    Hitos importantes en la evolución de HTML

  • 1991: HTML 1.0: Tim Berners-Lee publica el primer documento oficial que describe HTML, con un conjunto básico de etiquetas para estructurar documentos, incluyendo encabezados, párrafos, formato de texto, listas e hipervínculos.
  • 1995: HTML 2.0: Estandariza características que ya estaban siendo ampliamente utilizadas, como los formularios.
  • 1997: HTML 3.2: Se introducen etiquetas más avanzadas, como las tablas, y se mejora el soporte para scripts y estilos de presentación.
  • 1997-1999: HTML 4.0 y 4.01: Representa un gran avance al promover la separación entre el contenido (HTML) y la presentación (CSS), y añade mejoras en formularios y el elemento < iframe>.
  • 2000: XHTML: Se establece un estándar de codificación más estricto basado en XML para mejorar la compatibilidad entre navegadores.
  • 2014: HTML5: Es un hito fundamental que incluye:

    • Nuevas etiquetas semánticas (por ejemplo, < article>, < nav>) para describir mejor el contenido.

  • Soporte nativo para audio y video, eliminando la necesidad de plugins como Flash.
  • Gráficos con las etiquetas < canvas> y SVG.
  • Mejoras en la interactividad, formularios y manejo de aplicaciones web complejas.
  • para diseño web responsivo (adaptable a diferentes dispositivos).

  • 2014-Actualidad: Continúa la evolución con versiones como HTML 5.3 y se sigue trabajando en la mejora continua, con un enfoque en la accesibilidad y la semántica.
